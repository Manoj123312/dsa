2) Time and Space Complexity

Common time complexities: O(1), O(log n), O(n), O(n log n), O(n²).

public static int findMax(int[] arr) {
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > max) max = arr[i];
    }
    return max;
}

How it works: scans all elements once → O(n), extra space O(1).

4) Strings

Strings are immutable in Java. Use StringBuilder for modifications.

public static String reverse(String s) {
    char[] ch = s.toCharArray();
    int l = 0, r = ch.length - 1;
    while (l < r) {
        char t = ch[l];
        ch[l] = ch[r];
        ch[r] = t;
        l++; r--;
    }
    return new String(ch);
}

13) Graphs (BFS/DFS)

public static void bfs(List<List<Integer>> g, int src) {
    boolean[] vis = new boolean[g.size()];
    Queue<Integer> q = new LinkedList<>();
    q.offer(src);
    vis[src] = true;
    while (!q.isEmpty()) {
        int u = q.poll();
        for (int v : g.get(u)) {
            if (!vis[v]) {
                vis[v] = true;
                q.offer(v);
            }
        }
    }
}

17) Dynamic Programming

public static int fibDP(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++) dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}

21) Interview Questions with Answers

  1. Array vs Linked List? Arrays O(1) indexing, linked list easier insertion/deletion at known node.
  2. Why binary search O(log n)? It halves search space each step.
  3. HashMap use case? Fast key-value lookup and frequency counting.
  4. BFS vs DFS? BFS level-order, DFS depth-order.
  5. Why DP? Avoid repeated subproblem computation.
  6. Quick sort worst case? Bad pivots lead to O(n²).
  7. Collision in HashMap? Multiple keys in same bucket.
  8. What is balanced tree? Height near log n for fast operations.
  9. Stack overflow? Too deep recursion / missing base case.
  10. Topological sort? Ordering for DAG dependencies.
